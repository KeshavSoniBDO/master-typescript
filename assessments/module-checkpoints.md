# Module Checkpoints

Use this file after each module assignment.

## Module 1 Checkpoint (Foundations)

1. Explain runtime vs compile-time in your own words.
2. Show one bug `unknown` prevents that `any` allows.
3. Write one exhaustive union handler using `never`.

## Module 2 Checkpoint (Objects/Functions)

1. Explain structural typing with one risk and one benefit.
2. Demonstrate excess property check behavior with literal vs variable.
3. Write a function contract using `keyof` and explain why it is safer.

## Module 3 Checkpoint (Generics)

1. Write a generic that preserves input/output relationship.
2. Add a useful constraint and explain why unconstrained version is unsafe.
3. Show one case where generic is over-engineering.

## Module 4 Checkpoint (Advanced Types)

1. Write one conditional type and explain distributive behavior.
2. Use `infer` to extract a nested type.
3. Build one mapped/template-literal utility for a real use case.

## Module 5 Checkpoint (Runtime Boundaries)

1. Decode unknown payload into domain model without blind assertions.
2. Show DTO -> domain mapping with at least one field transform.
3. Return structured validation errors.

## Module 6 Checkpoint (Tooling/Config)

1. Explain why each strict flag in this repo matters.
2. Propose a safe strictness migration plan for legacy files.
3. Define CI gates for typecheck quality.

## Module 7 Checkpoint (Architecture Patterns)

1. Model a workflow using discriminated unions.
2. Define a port and one adapter implementation.
3. Return explicit service result types for success/failure.

## Self-Review Rubric (0-2 per question)

- 0: unclear or incorrect
- 1: partly correct, weak trade-off reasoning
- 2: correct with clear reasoning and practical awareness

Target: average >= 1.5 before moving on.
