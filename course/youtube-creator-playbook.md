# YouTube Creator Playbook: Master TypeScript

## Purpose

Turn this repository into a structured, high-retention YouTube course without diluting technical depth.

## Series Positioning

Title format:

- Master TypeScript: From Foundations to Architecture (8 Modules)

Audience:

- JS developers moving to serious TypeScript
- working developers who know syntax but want design-level skill
- instructors building internal training tracks

## Season Structure

1. Episode 0: Course map and setup (10-15 min)
2. Episode block M1-M7: concept + gotchas + challenge pattern
3. Episode capstone: full-system build walkthrough

Recommended length per module video:

- primary lesson: 25-40 min
- gotchas deep-dive: 15-25 min
- assignment walkthrough: 20-35 min

## Recording Blueprint Per Module

Use this 6-part flow in every module episode:

1. Hook (60-90s)
   - one painful real-world bug this module prevents
2. Mental model
   - one whiteboard model, one sentence summary
3. Live code section
   - walk through gotcha file first
4. Challenge section
   - solve one exercise live without peeking
5. Design reflection
   - when this pattern is overkill
6. CTA
   - module assignment + checkpoint prompt

## Differentiation Rule

To avoid repetitive content:

- lesson explains principles
- gotchas shows failure modes
- exercises enforce transfer to new scenarios
- assignments simulate production
- checkpoints test explanation quality

Never reuse identical code across these layers.

## Retention Tactics

1. Open-loop question at minute 1 (answer at minute ~70% mark)
2. Intentional error demo and debugging narrative
3. Recap every 8-10 minutes with one sentence summary
4. On-screen "why this matters in production" tag for each segment

## Camera + Screen Choreography

- face cam for hook + concept framing
- code-only during implementation
- whiteboard or diagram mode for transitions and architecture states

## Asset Reuse Matrix

Source from repo:

- lesson: conceptual script base
- gotchas: short clips and Shorts content
- exercises: stream-style coding segments
- assignment: long-form project videos
- checkpoints: community challenge posts

## Shorts Strategy (from gotchas)

Create one Short per gotcha theme:

- "Type assertion is not validation"
- "Why optional is not undefined"
- "Discriminated union vs boolean flags"
- "How strict flags prevent hidden bugs"

Format:

1. bug in 5s
2. wrong fix in 5s
3. right fix in 20s
4. principle in 10s

## Publishing Cadence

- 2 long videos/week
- 3-5 Shorts/week
- 1 community post/week with checkpoint question

Suggested cadence:

- Tue: lesson video
- Thu: gotchas/exercise video
- Sat: assignment/capstone segment

## Community Engagement Loop

At the end of each module video, ask one concrete question:

- "Where is this bug hiding in your current project?"
- "Paste your decoder signature and I’ll review it"

Collect comments into next-episode Q and A segment.

## Quality Bar Before Publishing

Checklist per video:

- technical claims verified against repository code
- all commands run successfully
- no unexplained jumps in complexity
- one explicit trade-off discussion included
- one anti-pattern shown and corrected

## Monetizable Course Extensions

1. Advanced pack: architecture clinics (Modules 5-7)
2. Interview prep pack: checkpoint-to-interview conversion
3. Team training pack: instructor slides + workshop prompts

## Instructor Note

Your edge is not just syntax speed.

Your edge is teaching how to think:

- where TypeScript guarantees stop
- how to design boundaries
- how to keep systems maintainable under change
